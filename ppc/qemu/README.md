# PowerPC アセンブラ ワークショップ

PowerPC ベアメタル アセンブラの基礎から応用まで、12 のシナリオで段階的に学ぶ。

## 前提条件

| ツール | 用途                 | インストール例                |
| :---   | :---                 | :---                          |
| Clang  | PowerPC アセンブラ   | `apt install clang`           |
| LLD    | PowerPC リンカ       | `apt install lld`             |
| QEMU   | PowerPC エミュレータ | `apt install qemu-system-ppc` |

## ビルドと実行

```bash
# 全シナリオをビルド
make

# 特定のシナリオを QEMU で実行
make run S=s01_hello

# 全シナリオを順に実行
make runall

# 逆アセンブル
make dump S=s01_hello

# 掃除
make clean
```

## ハードウェア情報

### QEMU bamboo マシン

- **CPU**: PowerPC 405GP
- **メモリ**: 64MB RAM
- **UART**: 16550 compatible at 0xEF600300
- **タイマー**: Time Base register（特殊レジスタ）

### 16550 UART レジスタ

| オフセット | 名称 | 方向 | 説明                                    |
| :---       | :--- | :--- | :---                                    |
| 0x00       | RBR  | R    | Receiver Buffer Register（受信データ）   |
| 0x00       | THR  | W    | Transmit Holding Register（送信データ） |
| 0x05       | LSR  | R    | Line Status Register（状態レジスタ）     |

**LSR ビットフラグ**:

| ビット | 名称 | 説明                                          |
| :---   | :--- | :---                                          |
| bit 0  | DR   | Data Ready — 受信データあり（1=受信完了）      |
| bit 5  | THRE | Transmitter Holding Register Empty — 送信可能（1=書き込み可） |

**受信手順**: LSR bit 0（DR）が 1 になるまでポーリング → RBR（0x00）から 1 バイト読み取り

**送信手順**: LSR bit 5（THRE）が 1 になるまでポーリング → THR（0x00）へ 1 バイト書き込み

### 終了方法

PowerPC ベアメタルでは標準的な終了処理が存在しない。QEMU は無限ループで停止を検出できないため、`--foreground 1` オプションで 1 秒後にタイムアウトさせる。

## I/O ルーチン

PowerPC ベアメタル（QEMU `bamboo` マシン）では、**プログラムが呼び出せる外部 I/O コール（BIOS・システムコール・セミホスティング・終了デバイス等）は提供されない**。すべての入出力と終了はハードウェアレジスタ（MMIO）の直接操作か、CPU の停止（無限ループ）による。

| 操作         | 方法                         | 参照先                                                         |
| :---         | :---                         | :---                                                           |
| 文字出力     | 16550 UART MMIO（`0xEF600300`） | [16550 UART レジスタ](#16550-uart-レジスタ) の THR 書き込み |
| 文字入力     | 16550 UART MMIO              | 同上の RBR / LSR ポーリング                                     |
| 時刻取得     | Time Base レジスタ（SPR）    | S08 を参照                                                     |
| プログラム終了 | 無限ループ + QEMU タイムアウト | [終了方法](#終了方法)                                        |

**他アーキテクチャとの対比**: ARM64（semihosting）、RISC-V（SiFive test device）、x86（BIOS 割り込み）、6502（cc65 ランタイム）、Z80（CP/M BDOS）のように、QEMU・ROM・OS が提供する終了デバイスやシステムコールは **`bamboo` マシンでは利用できない**。これは教材が選んだ QEMU マシン設定の制約であり、PowerPC というアーキテクチャ自体の制限ではない（他の PPC マシン・実機環境では Open Firmware 等が提供されることがある）。

**定義元**: 該当なし（外部コール ABI は存在しない）。I/O は 16550 UART の MMIO レジスタ仕様に従う。

## トラブルシューティング

### QEMU が起動しない

```bash
# 必要なツールがインストールされているか確認
which clang ld.lld qemu-system-ppc
# すべてのパスが表示されれば OK
```

- `clang` が見つからない → `apt install clang`（macOS では Xcode Command Line Tools）
- `ld.lld` が見つからない → `apt install lld`
- `qemu-system-ppc` が見つからない → `apt install qemu-system-ppc`（macOS では `brew install qemu`）

### ビルドエラーが発生する

`--target=powerpc-none-elf` が正しく指定されているか確認する。Makefile 内の `ASFLAGS` をチェック:

```makefile
ASFLAGS = --target=powerpc-none-elf -c
```

リンカスクリプト `linker.ld` がカレントディレクトリに存在することを確認。ない場合は `make clean` を実行してから再ビルドする。

### UART が動作しない（文字が表示されない）

1. **アドレス確認**: UART のベースアドレスが `0xEF600300` であることを確認
2. **マシンタイプ確認**: QEMU 起動時に `-machine bamboo` フラグが指定されているかチェック
3. **LSR ポーリング**: 送信前に必ず LSR bit 5（THRE）が 1 になるまでポーリングしているか確認
4. **レジスタ幅**: `STB`（Store Byte）命令で 1 バイト書き込みを行っているか確認。`STW`（Store Word）を使うと隣接レジスタを破壊する可能性がある
