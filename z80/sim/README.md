# Z80 アセンブリ学習教材 (CP/M シミュレータ)

Go 製 Z80 シミュレータ（simz80）上で動作する Z80 アセンブリ言語教材。CP/M BDOS 呼び出し規約を I/O に使い、全 13 のシナリオで段階的に学ぶ。

## 前提条件

| ツール   | 用途                                | インストール                 |
| :---     | :---                                | :---                         |
| `z80asm` | アセンブラ                          | `brew install z80asm`        |
| `simz80` | Z80 + CP/M シミュレータ（Go 製）    | `go build -o simz80 main.go` |
| Go       | simz80 ビルド用                     | `brew install go`            |

## ビルドと実行

```bash
# 全シナリオをビルド
make

# 特定のシナリオを simz80 で実行
make run S=s01_hello

# 全シナリオを連続実行（timeout 1 秒付き）
make runall

# 逆アセンブル（hex dump）
make dump S=s01_hello

# CP/M 互換ファイルを cpm/ に生成（yaze-ag 用）
make cpm

# 掃除
make clean
```

### yaze-ag（CP/M 3.1 エミュレータ）での実行

[yaze-ag](https://github.com/andreasgerlich/yaze-ag) で実際の CP/M 環境上で .com を実行できる。

```bash
# 1. yaze-ag をインストール
brew install yaze-ag

# 2. yaze-ag ディスクイメージを ~/cpm/ に配置（初回のみ）
make cpm-setup

# 3. CP/M 互換ファイルをビルド
make cpm
```

CP/M のファイル名に `_` は使えないため、`make cpm` はアンダースコアを除去したコピーを `cpm/` に生成する：

| simz80 用          | CP/M 用 (cpm/)   |
| :---               | :---             |
| `s01_hello.com`    | `S01HELLO.com`   |
| `s02_registers.com`| `S02REGS.com`    |
| `s12_minishell.com`| `S12MSHL.com`    |

`.yazerc` 設定:

```text
mount a <yaze-ag-disks>/BOOT_UTILS.ydsk
mount b <yaze-ag-disks>/CPM3_SYS.ydsk
mount c <repo-root>/z80/sim/cpm
go
```

実行例:

```text
$ yaze

A>c:s01
Hello, Z80 World!

A>e
```

## シナリオ一覧

### S01: Hello World

**難易度**: ★☆☆☆☆

最も基本的な CP/M プログラム。BDOS 関数 9（文字列出力）を使い、`$` 終端文字列をコンソールに出力する。

| 学習項目       | 説明                              |
| :---           | :---                              |
| `org 0x0100`   | CP/M プログラムの標準開始アドレス  |
| `ld c, 9`      | BDOS 関数番号を C レジスタに設定  |
| `ld de, msg`   | 文字列アドレスを DE レジスタに設定 |
| `call 0x0005`  | BDOS 呼び出し（CP/M システムコール）|
| `ret`          | CP/M に復帰（SP に積まれた 0x0000 に戻りシミュレータ停止）|
| `defm`         | 文字列定数の定義                  |
| `$` 終端       | BDOS fn 9 の文字列終端記号        |

**終了方法**: `RET` で CP/M に復帰（スタック上の 0x0000 に戻ると simz80 が停止）

> **コラム: CP/Mが作った$終端文字列の慣習──なぜnullじゃないのか**
>
> このリポジトリのZ80プログラムでは文字列を `$` で終端している。これはCP/MのBDOS関数9の規約による。
> Unix系のヌル終端（`\0`）やDOSとは異なり、CP/MはIntelのPL/Mコンパイラの文字列表現（`'...$'`）を継承したため
> `$` を終端記号に選んだと言われる。この慣習は後にMS-DOSの `INT 21h, fn 9` にも引き継がれた。

### S02: レジスタと算術

**難易度**: ★☆☆☆☆

Z80 の 8-bit レジスタ（A, B, C, D, E, H, L）と 16-bit レジスタペア（BC, DE, HL）の基本操作を学ぶ。16 進数表示サブルーチンも実装する。

| 学習項目           | 説明                                      |
| :---               | :---                                      |
| `ld a, n` / `ld r, r` | 即値・レジスタ間のデータ転送           |
| `add a, r`         | 8-bit 加算：A = A + r                     |
| `dec r` / `inc r`  | デクリメント・インクリメント             |
| `ld bc, nn`        | 16-bit 即値ロード                        |
| `add hl, bc`       | 16-bit 加算：HL = HL + BC                 |
| `inc hl`           | 16-bit インクリメント（0x00FF → 0x0100）  |
| `rrca` ×4          | A を右ローテート 4 回（上位ニブル取得）   |
| `and 0x0F`         | 下位ニブルのマスク抽出                   |
| `cp 10` → `jr c`   | 比較＋条件分岐（数字 vs A-F の判定）     |

**ポイント**: Z80 は `INC BC` / `DEC BC` も可能で、8-bit 同様に 16-bit のインクリメント・デクリメントが使える。

---

### S03: スタック操作

**難易度**: ★★☆☆☆

スタック（LIFO）の動作を理解し、`PUSH` / `POP` によるレジスタの保存・復元を学ぶ。ネストした PUSH/POP も実践する。

| 学習項目         | 説明                                          |
| :---             | :---                                          |
| `push af`        | AF レジスタペアをスタックに PUSH（SP -= 2）   |
| `push bc`        | BC レジスタペアをスタックに PUSH              |
| `pop bc`         | スタックから BC に POP（SP += 2）             |
| `pop af`         | スタックから AF に POP                        |
| LIFO             | Last In, First Out（最後に積んだものが最初に戻る）|
| ネスト PUSH/POP  | PUSH DE → PUSH HL → POP DE → POP DE で値の入れ替わりを確認 |

**ポイント**: Z80 では AF も PUSH/POP 可能。`PUSH AF` → `POP BC` のように異なるレジスタペアでも POP できる（値の移動に使える）。

---

### S04: ループ

**難易度**: ★★☆☆☆

`DJNZ`（デクリメント＋非ゼロ分岐）、`CP` + `JR NZ`（カウントアップ）、ネストループの 3 種類の反復処理を学ぶ。

| 学習項目                  | 説明                                      |
| :---                      | :---                                      |
| `djnz label`              | B--; B≠0 なら label へ相対ジャンプ        |
| `ld b, n` → `djnz`       | カウントダウンループの定石                |
| `cp n` → `jr nz, label`  | 比較＋非ゼロ分岐（カウントアップループ）  |
| ネストループ              | 外側ループ（C）+ 内側ループ（DJNZ）        |
| `push bc` / `pop bc`      | BDOS 呼び出し前後のカウンタ退避・復元     |

**ポイント**: BDOS 呼び出しは B レジスタを破壊するため、ループ内で BDOS を呼ぶ場合は `PUSH BC` / `POP BC` でカウンタを保護する必要がある。

---

### S05: 文字列操作

**難易度**: ★★☆☆☆

文字列の長さ測定（strlen）、`LDIR` によるブロックコピー（strcpy）、大文字変換を実装する。

| 学習項目             | 説明                                        |
| :---                 | :---                                        |
| `ld a, (hl)`         | HL の指すアドレスから 1 バイト読み込み      |
| `inc hl` / `inc de`  | ポインタのインクリメント                    |
| `cp '$'` → `jr z`    | 文字列終端（$）の検出                       |
| `cp 'a'` / `cp 'z'+1`| 範囲チェックによる小文字判定                |
| `sub 32`             | 小文字→大文字変換（ASCII 'a' - 'A' = 32）  |
| `ldir`               | ブロック転送：[DE]←[HL], HL++, DE++, BC--（BC>0 の間繰り返し）|
| `defs 32`            | 未初期化バッファ領域の確保                  |

**ポイント**: CP/M 環境では `$` を文字列終端に使う慣習がある（`defm` の BDOS fn 9 規約）。`LDIR` は Z80 の高速ブロック転送命令で、`LDI` を BC 回繰り返す。

---

### S06: シリアル入力

**難易度**: ★★★☆☆

BDOS 関数 1（コンソール入力）を使い、キーボードから文字を読み込む対話型プログラム。Enter キーで終了する。

| 学習項目           | 説明                                  |
| :---               | :---                                  |
| `ld c, 1`          | BDOS fn 1: コンソール入力（ブロッキング）|
| `call 0x0005`      | A レジスタに入力文字が返る            |
| `cp 13`            | CR（キャリッジリターン）判定          |
| `cp 10`            | LF（ラインフィード）判定              |
| `jr z, label`      | ゼロフラグによる条件分岐              |
| 入力ループ         | 文字受信 → 判定 → 継続/終了 のループ  |

**入力処理の流れ**:

1. `ld c, 1` で BDOS fn 1 を指定
2. `call 0x0005` でキー入力待ち（ブロッキング）
3. A レジスタに受信文字が格納される（BDOS が自動エコーバック）
4. CR/LF ならループ終了、それ以外は継続

### S07: サブルーチン

**難易度**: ★★★☆☆

`CALL` / `RET` による関数呼び出しと、レジスタ渡し・スタック渡しのパラメータ受け渡しを学ぶ。IX を使ったスタックフレームアクセスも実践する。

| 学習項目                    | 説明                                          |
| :---                        | :---                                          |
| `call label`                | サブルーチン呼び出し（戻りアドレスをスタックに PUSH）|
| `ret`                       | スタックから戻りアドレスを POP して復帰       |
| レジスタ渡し                | A, B レジスタで引数を受け渡し                 |
| ネスト CALL                 | サブルーチン内からさらにサブルーチンを呼ぶ    |
| `push` / `pop` による保存   | 呼び出し先でのレジスタ退避・復元             |
| `ld ix, 0` → `add ix, sp`  | IX をスタックフレームポインタとして使用       |
| `ld e, (ix+4)`              | スタック上の引数に IX+オフセットでアクセス   |

**Z80 サブルーチン呼び出し規約**:

| 項目       | 規約                                      |
| :---       | :---                                      |
| 引数渡し   | レジスタ（A, BC, DE, HL）またはスタック   |
| 戻り値     | A（8-bit）または HL（16-bit）             |
| 保存義務   | 呼び出し先で必要なレジスタを PUSH/POP で保存 |

### S08: ハードウェアアクセス

**難易度**: ★★★☆☆

BDOS 関数 12（CP/M バージョン番号取得）と、Z80 の `IN` / `OUT` 命令による I/O ポートアクセスを学ぶ。

| 学習項目           | 説明                                        |
| :---               | :---                                        |
| `ld c, 12`         | BDOS fn 12: CP/M バージョン番号取得         |
| `call 0x0005`      | HL にバージョン番号が返る（H=major, L=minor）|
| `out (n), a`       | 即値ポート n に A の値を出力                |
| `in a, (n)`        | 即値ポート n から A に読み込み              |
| `in a, (c)`        | レジスタ C で指定したポートから読み込み     |
| ポートアドレッシング | 即値（0x00-0xFF）または C レジスタ間接     |

**ポイント**: 本シミュレータでは I/O ポートはフローティングバスを模擬して 0xFF を返す。実機では各ポートが特定のハードウェアに対応する。

### S09: 条件分岐

**難易度**: ★★★★☆

`CP` 命令と各種条件分岐（`JR C/NC/Z/NZ`）を使った複数条件チェーンと、分岐テーブルパターンを学ぶ。

| 学習項目             | 説明                                      |
| :---                 | :---                                      |
| `cp n`               | A - n の減算結果をフラグに反映（A は不変）|
| `jr c, label`        | キャリーフラグ（C=1）で相対ジャンプ       |
| `jr nc, label`       | キャリーなし（C=0）で相対ジャンプ         |
| `jr z, label`        | ゼロフラグ（Z=1）で相対ジャンプ           |
| `jr nz, label`       | 非ゼロ（Z=0）で相対ジャンプ               |
| 複数条件チェーン     | CP → JR Z/NZ を連鎖させた if-else 構造    |
| 分岐テーブル         | CP → JR Z のチェーンでインデックス分岐    |
| `ex de, hl`          | DE ↔ HL の交換（print_str の引数切り替え） |

**CP 命令のフラグ変化**:

| 条件         | フラグ               | 使用する JR    |
| :---         | :---                 | :---           |
| A == n       | Z=1                  | `jr z`         |
| A != n       | Z=0                  | `jr nz`        |
| A < n（符号なし）| C=1              | `jr c`         |
| A >= n（符号なし）| C=0             | `jr nc`        |

### S10: ビット演算

**難易度**: ★★★★☆

`AND` / `OR` / `XOR` / `CPL` の論理演算と、`SLA` / `SRA` / `SRL` のシフト命令、`BIT` / `SET` / `RES` のビット操作を学ぶ。

| 学習項目         | 説明                                          |
| :---             | :---                                          |
| `and n`          | ビット単位の AND（マスク抽出）                |
| `or n`           | ビット単位の OR（ビット結合）                |
| `xor n`          | ビット単位の XOR（ビット反転）               |
| `cpl`            | A の全ビット反転（1 の補数）                  |
| `sla a`          | 左算術シフト（bit 7 → C, LSB=0）             |
| `sra a`          | 算術右シフト（MSB 維持、符号保持）            |
| `srl a`          | 論理右シフト（MSB=0）                         |
| `set n, r`       | レジスタの bit n を 1 にセット               |
| `res n, r`       | レジスタの bit n を 0 にクリア               |
| `bit n, r` → `jr nz` | ビットテスト（Z フラグに結果反映）        |

**応用例**:

- `and 0x0F`: 下位 4 ビットの抽出（ニブルマスク）
- `or 0x80`: 最上位ビットのセット
- `xor 0xFF`: 全ビット反転（2 の補数ではない）
- `sla a`: ×2 の高速乗算

### S11: メモリ操作

**難易度**: ★★★★☆

メモリへのブロック単位の書き込み（fill）、`LDIR` によるコピー、`CPIR` による検索、IX/IY を使ったインデックスアドレッシングを学ぶ。

| 学習項目              | 説明                                            |
| :---                  | :---                                            |
| `ld (hl), a` → `djnz` | ループによるメモリ fill の定石                  |
| `ldir`                | ブロック転送（HL→DE, BC バイト分）              |
| `cpir`                | ブロック検索（HL から BC バイト分、A の値を検索）|
| `ld ix, addr`         | IX に 16-bit アドレスをロード                   |
| `ld a, (ix+3)`        | IX + 3 のアドレスから読み込み                   |
| `ld (iy+3), a`        | IY + 3 のアドレスに書き込み                     |
| `defs 16`             | 16 バイトの未初期化バッファ確保                 |

**CPIR の動作**:

1. A と (HL) を比較
2. HL++, BC--（一致しなければ 1 に戻る）
3. 一致または BC=0 で終了（HL は一致バイトの次を指す）

### S12: ミニシェル（総合）

**難易度**: ★★★★★

BDOS 関数 10（バッファ入力）を使った対話型コマンドシェル。文字列比較によるコマンド解析と、REPL ループを実装する。

| 学習項目                | 説明                                      |
| :---                    | :---                                      |
| BDOS fn 10              | コンソール行入力（バッファ読み取り）      |
| 入力バッファ構造        | `[最大長] [実際の長さ] [文字列...]`       |
| `strcmp` 実装           | `$` 終端文字列の比較ルーチン              |
| コマンドディスパッチ    | strcmp → JR Z チェーンによるコマンド分岐  |
| REPL                    | Read-Eval-Print Loop（プロンプト→入力→実行→ループ）|
| `or 1` / `xor a`        | フラグ操作による Z フラグのセット/クリア  |

**コマンド一覧**:

- `help` — コマンド一覧を表示
- `hello` — 挨拶メッセージを表示
- `regs` — 現在のレジスタ値（A, B）を表示
- `quit` / `q` — シェルを終了

**BDOS fn 10 のバッファ構造**:

```text
inbuf:
    db  32       ; +0: 最大入力文字数（バッファサイズ-1）
    db  ?        ; +1: BDOS が実際の入力文字数を格納
    ds  32       ; +2: 入力文字列が格納される領域
```

## 拡張シナリオ

### S13: 裏レジスタ（Shadow Registers）

**難易度**: ★★☆☆☆

Z80 独自の裏レジスタ（alternate register set）と、`EXX` / `EX AF,AF'` による高速コンテキスト切り替えを学ぶ。

Z80 は「メイン」と「裏（shadow）」の 2 組のレジスタを持つ：

```text
 メイン:  A   F   B   C   D   E   H   L
   裏:  A'  F'  B'  C'  D'  E'  H'  L'
```

| 学習項目                | 説明                                          |
| :---                    | :---                                          |
| `exx`                   | BC↔BC', DE↔DE', HL↔HL' を同時に入れ替え（1 命令）|
| `ex af, af'`            | AF↔AF' を入れ替え                             |
| ISR コンテキスト保存    | EXX + EX AF,AF' の 2 命令で全レジスタ退避     |
| ISR コンテキスト復元    | EXX + EX AF,AF' の 2 命令で全レジスタ復元     |

**従来の PUSH/POP との比較**:

| 方法            | 命令数 | スタック使用量 | 実行速度 |
| :---            | :---   | :---           | :---     |
| PUSH/POP × 4    | 8 命令 | 8 バイト       | 遅い     |
| EXX + EX AF,AF' | **2 命令** | **0 バイト** | **高速** |

**注意**: IX, IY, SP には裏レジスタがない。また、裏レジスタは割り込みハンドラ（ISR）での高速コンテキスト保存が主な用途であり、通常のプログラミングでは使用に注意が必要。

## I/O ルーチン

本教材は Z80 実機ではなく、**教材独自の CP/M シミュレータ `simz80`（`z80/sim/cmd/simz80/main.go`）** 上で動作する。`simz80` は CP/M 2.2 の BDOS（Basic Disk Operating System）呼び出しを **互換サブセットとして再実装** したものであり、実機 CP/M そのものではない点に注意。I/O はすべて BDOS 関数呼び出し（`CALL 0x0005`）で行い、UART などのハードウェアを直接操作しない。

**呼出規約**:

| 項目     | 説明                                                                |
| :---     | :---                                                                |
| 呼び出し | レジスタ C に関数番号を設定し `CALL 0x0005`                         |
| 引数     | 関数により E または DE レジスタ                                     |
| 戻り値   | A または HL（関数による）                                           |
| 終了     | `RET` でスタック上の `0x0000` に戻ると `PC=0x0000` をトラップして停止 |

**BDOS 関数（`simz80` 実装状況付き）**:

| 関数番号 (C) | 機能               | 入力                   | 戻り値                                     | simz80 | 教材使用   |
| :---         | :---               | :---                   | :---                                       | :---   | :---       |
| 1            | コンソール入力     | なし                   | A = 入力文字（`& 0x7F`、エコー付き）       | 実装   | S06, S12   |
| 2            | コンソール出力     | E = 文字               | なし（`E & 0x7F` を出力）                  | 実装   | 全シナリオ |
| 6            | ダイレクトコンソール I/O | E = 文字 / `0xFF`（入力要求） | A = 文字 or 0（非ブロッキング） | **未実装**（A=`0xFF`） | 未使用 |
| 9            | 文字列出力         | DE = 文字列アドレス    | なし（`'$'` または NUL まで出力　※）       | 実装   | S01        |
| 10           | コンソール行入力   | DE = バッファアドレス  | バッファに格納（`buf[1]`=長さ）            | 実装   | S12        |
| 12           | バージョン番号取得 | なし                   | B=`0x00`, A=`0x22`, HL=`0x0022`（CP/M 2.2）| 実装   | S08        |
| その他       | —                  | —                      | A = `0xFF`（未実装関数の既定動作）         | 未実装 | —          |

※ 公式 CP/M の文字列終端は `'$'`（0x24）のみ。`simz80` は `'$'` と NUL（0x00）の両方を終端として扱う。

**fn 10 のバッファ形式**（`DE` が指すアドレス）:

| オフセット | 内容                                 |
| :---       | :---                                 |
| `buf[0]`   | 最大長（入力前にプログラムが設定）   |
| `buf[1]`   | 実際の入力長（`simz80` が設定）      |
| `buf[2..]` | 入力文字列（`simz80` が設定）        |

**BIOS 層について**: 実機 CP/M は BDOS（高レベル OS API）と BIOS（低レベルデバイスドライバ）の 2 層構造で、BIOS は `0x0000` のジャンプベクタ経由でアクセスする。本教材の `simz80` は **BIOS 層を実装せず、BDOS のみ** を提供する。通常の I/O は BDOS 経由で十分である。

**プログラム構造と終了**:

```asm
org 0x0100          ; CP/M の TPA（Transient Program Area）開始アドレス

_start:
    ; ... プログラム本体 ...
    ret             ; スタック上の 0x0000 に戻り PC=0x0000 をトラップ → 終了
```

`simz80` はプログラム起動時にスタックへ終了アドレス `0x0000` を積む。`_start` から `ret` すると `PC=0x0000` となり、これをトラップしてシミュレータが終了する。

**定義元**: 教材独自 `simz80`（`z80/sim/cmd/simz80/main.go` の `bdosCall()`、fn 1/2/9/10/12 のみ実装）。実機 CP/M BDOS 仕様とは細部（終端文字・未実装関数の扱い）が異なる。
