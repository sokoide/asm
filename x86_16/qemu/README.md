# 8086 アセンブラ ワークショップ

x86 リアルモード（16-bit）からプロテクトモード（32-bit）まで、14 のシナリオで段階的に学ぶ。

## 前提条件

| ツール | 用途                 | インストール例                |
| :---   | :---                 | :---                          |
| NASM   | アセンブラ           | `apt install nasm`            |
| LD     | リンカ（elf32 対応） | GNU binutils に同梱           |
| QEMU   | エミュレータ         | `apt install qemu-system-x86` |

## ビルドと実行

```bash
# 全シナリオをビルド
make

# 特定のシナリオを QEMU で実行
make run S=s01_hello

# 逆アセンブル（16-bit 用）
make dump S=s01_hello

# 逆アセンブル（32-bit セクタ2以降、S13/S14 用）
make dump32 S=s14_protected

# 掃除
make clean
```

## シナリオ一覧

### S01: Hello World

**難易度**: ★☆☆☆☆

最も基本的なブートセクタプログラム。画面に文字列を表示して停止する。

| 学習項目          | 説明                                                 |
| :---              | :---                                                 |
| `bits 16`         | 16-bit リアルモードでアセンブル                      |
| セグメント初期化  | DS / ES / SS を 0 に設定                             |
| `INT 0x10 AH=0Eh` | BIOS テレタイプ出力（1文字ずつ表示）                 |
| `LODSB`           | `[DS:SI]` から 1 バイト読み込み、SI をインクリメント |
| `0xAA55`          | ブートシグネチャ（MBR の末尾 2 バイト）              |
| `CLI` + `HLT`     | 割り込み禁止 → CPU 停止                              |

**QEMU 終了方法**: `Ctrl+A` → `X`

### S04: ループと条件分岐

| 学習項目      | 説明                                          |
| :---          | :---                                          |
| `LOOP`        | CX を自動デクリメントし、CX≠0 ならジャンプ    |

### S05: 文字列操作

**難易度**: ★★☆☆☆

8086 の文字列命令群と `REP` プレフィックスを学ぶ。

| 学習項目        | 説明                                           |
| :---            | :---                                           |
| `LODSB`         | `[DS:SI]` → AL、SI++                           |
| `STOSB`         | AL → `[ES:DI]`、DI++                           |

| `REP` / `REPNE` | CX 回繰り返し（等しい/等しくない間）           |
| `CLD`           | ディレクションフラグをクリア（SI/DI 増加方向） |
| 文字列長測定    | `REPNE SCASB` で null ターミネータを探す       |

### S08: 直接 VRAM 書き込み

**難易度**: ★★★☆☆

BIOS を経由せず、ビデオメモリ（`0xB800`）に直接書き込んでカラー文字を表示する。

| 学習項目          | 説明                                             |
| :---              | :---                                             |
| VRAM セグメント   | `ES = 0xB800`（テキストモード 80×25）            |
| キャラクタ + 属性 | `[ASCII][属性]` の 2 バイトで 1 文字を構成       |
| 属性バイト        | `[Blink][BG RGB][Bright][FG RGB]`（計 8 ビット） |
| `STOSW`           | AX（文字+属性）を `[ES:DI]` に書き込み           |
| カラーパレット    | 16 色の前景色・背景色の組み合わせ                |

### S09: 分岐とメニュー

**難易度**: ★★★★☆

キー入力に応じた分岐でメニューシステムを構築する。符号付き / 符号なし比較の違いも体験。

| 学習項目              | 説明                                     |
| :---                  | :---                                     |
| `CMP` とフラグ        | ZF, CF, SF, OF の変化を観察              |
| 符号付き: `JL` / `JG` | SF ≠ OF のときにジャンプ                 |
| 符号なし: `JB` / `JA` | CF = 1 のときにジャンプ                  |
| メニューディスパッチ  | キー入力 → `CMP` → `JE` で該当処理へ分岐 |

**ポイント**: `0xFB` は符号付きなら -5、符号なしでは 251。同じビット列でも比較命令により結果が変わる。

### S13: マルチセクタ読み込み

**難易度**: ★★★★★

512 バイトの壁を超え、ディスクから追加セクタを読み込む。プロテクトモードへの前提知識。

| 学習項目             | 説明                                               |
| :---                 | :---                                               |
| `INT 0x13 AH=02h`    | ディスクからセクタを読み込みメモリに格納           |
| CHS アドレッシング   | Cylinder / Head / Sector でディスク位置を指定      |
| セクタ番号           | 1 始まり（セクタ 2 = オフセット 512 バイト目）     |
| `0x7E00`             | ブートセクタ直後のメモリ（0x7C00 + 512）           |
| `ORG 0x7C00`         | NASM `-f bin` でアドレスを 0x7C00 に固定           |
| クロスセクタ呼び出し | セクタ 2 からセクタ 1 のサブルーチンを呼び出し可能 |

**512 バイトを超えるコードを書くための仕組み**:

```text
BIOS がセクタ 1 (0x7C00) を自動ロード
       ↓
自前のコードで INT 0x13 でセクタ 2+ を 0x7E00 へロード
       ↓
JMP 0x7E00 でセクタ 2 のコードへ移行
```

### S14: 32-bit プロテクトモード

**難易度**: ★★★★★

16-bit リアルモードから 32-bit プロテクトモードへ移行。BIOS が使えなくなり、全てハードウェア直接操作になる。

| 学習項目                      | 説明                                                                   |
| :---                          | :---                                                                   |
| GDT (Global Descriptor Table) | セグメントの属性（基地址・上限・権限）を定義                           |
| `LGDT`                        | GDT レジスタにテーブルの位置とサイズをロード                           |
| CR0.PE ビット                 | 0→1 でプロテクトモード有効化                                           |
| ファージャンプ                | `JMP 0x08:pm_entry` でパイプラインをフラッシュ                         |
| セグメントセレクタ            | `0x08` = コード, `0x10` = データ（GDT インデックス）                   |
| 32-bit レジスタ               | EAX, ESI, EDI, ESP（16-bit の拡張版）                                  |
| リニアアドレス                | VRAM = `0x000B8000`（リアルモードの `0xB800:0000` と同一物理アドレス） |
| フラットメモリモデル          | Base=0, Limit=4GB（全メモリに線形アクセス）                            |

**プロテクトモード移行の手順**:

```text
1. CLI           ← 割り込み禁止
2. LGDT [gdt]    ← GDT を CPU に登録
3. CR0 |= 1      ← PE ビットをセット
4. JMP 0x08:addr ← ファージャンプ（16-bit キューをフラッシュ）
5. セグメントレジスタに 0x10 を設定
   ↑ ここから 32-bit 命令が使える、BIOS は使用不可
```

## ビルド方式の違い

|              | S01〜S12                      | S13〜S14                             |
| :---         | :---                          | :---                                 |
| フォーマット | NASM `-f elf32` + リンカ      | NASM `-f bin`（直接出力）            |
| セクタ数     | 1（512 バイト）               | 2（1024 バイト）                     |
| アドレス指定 | リンカスクリプト（linker.ld） | `ORG 0x7C00`                         |
| 逆アセンブル | `make dump S=...`             | `make dump32 S=...`（セクタ 2 以降） |

## 主要命令リファレンス

### データ転送

| 命令           | 動作                              |
| :---           | :---                              |
| `MOV dst, src` | dst ← src                         |
| `PUSH src`     | スタックに保存（SP -= 2）         |
| `POP dst`      | スタックから復元（SP += 2）       |
| `XCHG a, b`    | a と b を交換                     |
| `LEA reg, mem` | 実効アドレスを計算して reg に格納 |

### 算術

| 命令           | 動作                     |
| :---           | :---                     |
| `ADD dst, src` | dst += src               |
| `SUB dst, src` | dst -= src               |
| `INC dst`      | dst++                    |
| `DEC dst`      | dst--                    |
| `MUL src`      | AX ← AL × src（8-bit）   |
| `DIV src`      | AL ← AX ÷ src, AH ← 余り |

### 論理・シフト

| 命令           | 動作                      |
| :---           | :---                      |
| `AND dst, src` | ビット単位の AND          |
| `OR dst, src`  | ビット単位の OR           |
| `XOR dst, src` | ビット単位の XOR          |
| `NOT dst`      | ビット反転                |
| `SHL dst, 1`   | 左に 1 ビットシフト（×2） |
| `SHR dst, 1`   | 右に 1 ビットシフト（÷2） |

### 比較・分岐

| 命令          | 動作                      | 条件        |
| :---          | :---                      | :---        |
| `CMP a, b`    | a - b（フラグだけ更新）   | —           |
| `TEST a, b`   | a AND b（フラグだけ更新） | —           |
| `JMP label`   | 無条件ジャンプ            | —           |
| `JE` / `JZ`   | 等しいとき                | ZF=1        |
| `JNE` / `JNZ` | 等しくないとき            | ZF=0        |
| `JL` / `JNGE` | より小さい（符号付き）    | SF≠OF       |
| `JG` / `JNLE` | より大きい（符号付き）    | ZF=0, SF=OF |
| `JB` / `JNAE` | より小さい（符号なし）    | CF=1        |
| `JA` / `JNBE` | より大きい（符号なし）    | CF=0, ZF=0  |
| `LOOP label`  | CX--, CX≠0 ならジャンプ   | —           |

### 文字列

| 命令    | 動作                              |
| :---    | :---                              |
| `LODSB` | AL ← `[DS:SI]`, SI++              |
| `STOSB` | `[ES:DI]` ← AL, DI++              |
| `MOVSB` | `[ES:DI]` ← `[DS:SI]`, SI++, DI++ |
| `SCASB` | 比較: AL と `[ES:DI]`, DI++       |
| `REP`   | CX 回繰り返す                     |
| `REPNE` | 等しくない間、CX 回繰り返す       |

### 制御

| 命令         | 動作                             |
| :---         | :---                             |
| `CALL label` | 次のアドレスを PUSH してジャンプ |
| `RET`        | スタックから POP してジャンプ    |
| `CLI`        | 割り込み禁止                     |
| `STI`        | 割り込み許可                     |
| `HLT`        | CPU 停止                         |
| `NOP`        | 何もしない                       |

## BIOS 割り込みリファレンス

### INT 0x10 — ビデオサービス

| AH    | 機能                   | パラメータ                                          |
| :---  | :---                   | :---                                                |
| `00h` | ビデオモード設定       | AL = モード番号（3 = 80×25 テキスト）               |
| `01h` | カーソル形状設定       | CH, CL = 開始/終了スキャンライン（CH=20h で非表示） |
| `02h` | カーソル位置設定       | BH = ページ, DH = 行, DL = 列                       |
| `0Eh` | テレタイプ出力         | AL = 文字, BH = ページ                              |
| `13h` | 文字列表示（属性付き） | 詳細は BIOS 仕様書を参照                            |

### INT 0x16 — キーボードサービス

| AH    | 機能             | パラメータ                              |
| :---  | :---             | :---                                    |
| `00h` | キー入力待ち     | 戻り値: AL = ASCII, AH = スキャンコード |
| `01h` | キー入力チェック | 戻り値: ZF=0 で入力あり                 |

### INT 0x13 — ディスクサービス

| AH    | 機能             | パラメータ                                                                      |
| :---  | :---             | :---                                                                            |
| `02h` | セクタ読み込み   | AL=セクタ数, CH=シリンダ, CL=セクタ(1-), DH=ヘッド, DL=ドライブ, ES:BX=バッファ |
| `08h` | ドライブ情報取得 | DL=ドライブ                                                                     |

### プロテクトモード関連

| 命令           | 動作                                     |
| :---           | :---                                     |
| `LGDT [mem]`   | GDT の位置とサイズを GDTR にロード       |
| `MOV CR0, EAX` | CR0 制御レジスタに書き込み（bit 0 = PE） |
| `JMP far`      | セグメント:オフセット のファージャンプ   |

### I/O ポート

| ポート | 用途                                        |
| :---   | :---                                        |
| `0x64` | キーボードコントローラ（AL=FEh でリセット） |

## 次のステップ

- 各シナリオのソースを開き、コメントを読みながらコードを追う
- `make dump S=s02_registers` で機械語を確認し、命令がどうエンコードされるか観察する
- 値を変更してビルド → 実行し、結果の変化を確認する
- S12 をベースに新しいコマンド（例: `echo`, `date` 風のカウンタ）を追加してみる
