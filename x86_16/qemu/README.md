# 8086 アセンブラ ワークショップ（QEMU フロッピーブート版）

QEMU フロッピーディスクイメージからリアルモードで起動し、16550 UART (COM1, I/O 0x3F8) を I/O に使う x86 16-bit ベアメタルアセンブリ言語教材。14 のシナリオで段階的に学ぶ。

## 前提条件

| ツール           | 用途                 | インストール例                |
| :---             | :---                 | :---                          |
| NASM             | アセンブラ           | `apt install nasm`            |
| LD (elf_i386)    | リンカ               | GNU binutils に同梱           |
| QEMU (i386)      | エミュレータ         | `apt install qemu-system-x86` |

## ビルドと実行

```bash
# 全シナリオをビルド
make

# 特定のシナリオを QEMU で実行（1 秒のタイムアウト付き）
make run S=s01_hello

# 全シナリオを連続実行（S06/S12 は printf で入力自動送信）
make runall

# 逆アセンブル（16-bit コード）
make dump S=s01_hello

# 逆アセンブル（32-bit セクタ 2 以降、S13/S14 用）
make dump32 S=s14_protected

# 掃除
make clean
```

## シナリオ一覧

### S01: Hello World

**難易度**: ★☆☆☆☆

最も基本的なブートセクタプログラム。16550 UART (COM1, I/O 0x3F8) に `"Hello, 8086 World!"` を出力して停止する。

| 学習項目                     | 説明                                                       |
| :---                         | :---                                                       |
| `bits 16`                    | 16-bit リアルモードでアセンブル                            |
| セグメント初期化             | `DS`, `ES`, `SS` を 0 に設定                               |
| `mov sp, 0x7C00`             | スタックポインタをコード直下に配置                         |
| `lodsb`                      | `[DS:SI]` から 1 バイト読み込み、`SI` をインクリメント    |
| `uart_putc`（16550 ポーリング） | LSR (0x3FD) bit 5 (THRE) 待ち → THR (0x3F8) に書込み   |
| ブートシグネチャ             | `times 510-($-$$) db 0` + `dw 0xAA55`（512 バイトに収める） |
| `cli` + `hlt`                | 割り込み禁止 → CPU 停止（タイムアウトで QEMU 終了）       |

**QEMU 終了方法**: タイムアウト（`make run` は 1 秒後に強制終了）

### S02: レジスタと算術

**難易度**: ★☆☆☆☆

レジスタ間のデータ移動と四則演算を体験する。結果を 16 進数で表示するサブルーチンも導入。

| 学習項目                          | 説明                                            |
| :---                              | :---                                            |
| `mov`, `add`, `sub`, `inc`, `dec` | 基本算術命令                                    |
| `ah` / `al`                       | レジスタの上位・下位バイトへのアクセス          |
| レジスタコピー                    | `mov bx, ax` で値の複製                        |
| `print_hex16`                     | AX をニブル単位で ASCII 変換して 4 桁表示       |
| 16 進数表示（ニブル分解）         | `shr` + `and 0x0F` → 0-9/A-F の ASCII 化       |

**ポイント**: `mov ax, 0x1234` → `mov bx, ax` で BX も 0x1234 になる。

### S03: スタック操作

**難易度**: ★★☆☆☆

スタック（LIFO）の動作を理解する。PUSH/POP でデータを保存・復元するパターンを学ぶ。

| 学習項目             | 説明                                               |
| :---                 | :---                                               |
| `push` / `pop`       | スタックへの保存・復元（SP が ±2、16-bit 単位）    |
| LIFO                 | Last In, First Out（最後に入れたものが最初に出る） |
| レジスタ保存パターン | `push ax` → 作業 → `pop ax` で値を保護             |
| スタック成長方向     | 下方成長（SP が減少する）                          |
| `mov sp, 0x7C00`     | スタックポインタの初期化                           |

**ポイント**: `push 0x1234` → `push 0x5678` → `pop bx`（BX=0x5678）→ `pop ax`（AX=0x1234）で LIFO を実証。

### S04: ループと条件分岐

**難易度**: ★★☆☆☆

`LOOP` 命令と `CMP` + 条件ジャンプを使った反復処理。`TEST` 命令による偶数/奇数判定も学ぶ。

| 学習項目      | 説明                                          |
| :---          | :---                                          |
| `loop`        | CX を自動デクリメントし、CX≠0 ならジャンプ    |
| `cmp` + `jle` | 比較 → 条件付きジャンプ（カウントアップループ） |
| `test`        | AND を実行せずフラグだけ更新（偶数/奇数判定） |
| カウントダウン | `loop` で 5→1 を表示                         |
| カウントアップ | `inc cl` + `cmp cl, 6` + `jle` で 1→5 を表示 |

**ポイント**: `loop` は CX をデクリメントしてから判定する。CX=1 のときは 0 になってジャンプしない。

### S05: 文字列操作

**難易度**: ★★☆☆☆

8086 の文字列命令群と `REP` プレフィックスを学ぶ。文字列長測定、コピーを実装する。

| 学習項目        | 説明                                           |
| :---            | :---                                           |
| `lodsb`         | `[DS:SI]` → AL、SI++                          |
| `stosb`         | AL → `[ES:DI]`、DI++                          |
| `scasb`         | AL と `[ES:DI]` を比較、DI++                  |
| `rep` / `repne` | CX 回繰り返し（等しい/等しくない間）           |
| `cld`           | ディレクションフラグをクリア（SI/DI 増加方向） |
| `repne scasb`   | null 文字（AL=0）を検索して文字列長を測定     |
| `rep movsb`     | CX バイトを一括コピー                         |

**文字列長測定の手順**:
1. `mov di, greeting` — 検索開始アドレス
2. `xor al, al; mov cx, 0xFFFF` — null 検索
3. `repne scasb` — 検索実行
4. `0xFFFF - CX - 1` が文字数

### S06: シリアル入力

**難易度**: ★★★☆☆

16550 UART から文字を読み込み、エコーバックする対話型プログラム。Enter キーで終了。

| 学習項目              | 説明                                        |
| :---                  | :---                                        |
| `uart_getc`           | UART 受信サブルーチン                       |
| LSR (0x3FD) bit 0     | Data Ready — 1 = 受信データあり             |
| RBR (0x3F8) 読み込み  | `in al, dx` で受信データ取得                |
| ポーリングループ      | `in` → `test al, 0x01` → `jz` でデータ待ち |
| エコーバック          | 受信文字をそのまま `uart_putc` で出力       |
| Enter (CR/LF) 判定    | `cmp al, 13` / `cmp al, 10` で入力終了判定  |

**UART 受信の流れ**:
1. LSR (0x3FD) の bit 0 (Data Ready) をポーリング
2. Data Ready = 1 になったら RBR (0x3F8) から 1 バイト読み込み

### S07: サブルーチン

**難易度**: ★★★☆☆

`CALL` / `RET` による関数呼び出しと、レジスタ経由のパラメータ渡しを学ぶ。

| 学習項目         | 説明                                                |
| :---             | :---                                                |
| `call` / `ret`   | リターンアドレスをスタックに保存・復元              |
| パラメータ渡し   | 呼び出し前にレジスタ（AL, BL, CX, SI 等）に値を設定 |
| `pusha` / `popa` | 全汎用レジスタの一括保存・復元                      |
| ネスト呼び出し   | サブルーチンからさらに `call` で別の関数を呼ぶ     |
| `add_bytes`      | AL + BL → AL の計算サブルーチン                    |
| `print_dots`     | CX 個のドットを `loop` で出力（ネスト呼び出し例）  |

**`pusha` / `popa` の保存対象**: AX, CX, DX, BX, SP, BP, SI, DI（16-bit 8 個）

### S08: ハードウェアアクセス（8254 PIT）

**難易度**: ★★★☆☆

Intel 8254 PIT (Programmable Interval Timer) のカウンタ値を I/O ポート経由で読み取る。

| 学習項目               | 説明                                          |
| :---                   | :---                                          |
| PIT カウンタ 0         | I/O ポート 0x40（16-bit カウンタ値）          |
| PIT 制御レジスタ       | I/O ポート 0x43（カウンタラッチ指示）         |
| `out dx, al`           | 制御レジスタに `0x00` を書き込みカウンタ凍結  |
| `in al, dx`            | ローバイト → ハイバイトの順に読み取り         |
| 16-bit 値の再構成      | `mov bl, al`（lo）+ `mov bh, al`（hi）→ BX   |
| 2 回目の読み取り       | PIT が動作中（カウントダウン中）であることを確認 |

**PIT 読み取り手順**:
1. `mov al, 0x00` → `out 0x43, al` でカウンタ 0 をラッチ
2. `in al, 0x40` → BL（ローバイト）
3. `in al, 0x40` → BH（ハイバイト）

### S09: 条件分岐

**難易度**: ★★★★☆

`CMP` + 条件ジャンプを使った分岐処理を学ぶ。符号付き比較と符号なし比較の違いを体験する。

| 学習項目           | 説明                                          |
| :---               | :---                                          |
| `cmp`              | 2 つの値を比較（フラグのみ更新）              |
| `je` / `jne`       | ZF=1 で等しい / ZF=0 で等しくない             |
| `jl`（符号付き）    | SF≠OF のときジャンプ（less than）             |
| `jg`（符号付き）    | ZF=0 かつ SF=OF のときジャンプ（greater than） |
| 判定チェーン       | `cmp` → `jl` → `cmp` → `jg` で if-else 構築  |

**ポイント**: 42==42 → `je` で "Yes!"、-5 < 10 → `jl` で "Yes!"、30 > 10 → `jg` で "Yes!"。

### S10: ビット演算

**難易度**: ★★★★☆

AND / OR / XOR / SHL / SHR を使い、ビット単位の操作を理解する。2 進数表示サブルーチンも実装。

| 学習項目        | 説明                                                    |
| :---            | :---                                                    |
| `and`           | マスク抽出（`0x5A AND 0x0F = 0x0A`）                   |
| `or`            | ビット結合（`0x50 OR 0x0A = 0x5A`）                    |
| `xor`           | ビット反転（`0xFF XOR 0x0F = 0xF0`）                   |
| `xor ax, ax`    | レジスタのゼロ化（`mov ax, 0` より高速・短縮）          |
| `shl al, 1`     | 左論理シフト＝ ×2（`3 << 1 = 6`）                      |
| `shr al, 1`     | 右論理シフト＝ ÷2（`10 >> 1 = 5`）                     |
| `print_bin8`    | `shl` + キャリーフラグで 1 ビットずつ判定 → 2 進数表示 |

**2 進数表示の仕組み**: `shl al, 1` で最上位ビットをキャリーフラグに追い出し、`jc`/`jnc` で `'0'`/`'1'` を出力。

### S11: メモリとブロック操作

**難易度**: ★★★★☆

メモリへのブロック単位の書き込み・コピーと、8086 の多彩なアドレッシングモードを学ぶ。

| 学習項目          | 説明                                         |
| :---              | :---                                         |
| `rep stosb`       | メモリブロックのフィル（`.` で 20 バイト埋める） |
| `rep movsb`       | メモリブロックのコピー（`"Hello"` → バッファ）  |
| `[即値オフセット]` | `mov byte [display+3], 'L'` でインデックス指定 |
| `[BX+SI]`         | ベース + インデックスアドレッシング           |
| null 終端         | 書き込み後 `mov byte [di], 0` で手動終端      |

**アドレッシングモード**: `[display+0]`〜`[display+4]` の直接指定と `[bx+si]` の間接指定の両方を体験。

### S12: ミニシェル（総合）

**難易度**: ★★★★★

これまでの全要素（UART I/O、文字列比較、分岐、サブルーチン）を組み合わせた対話型コマンドシェル。

| 学習項目        | 説明                                          |
| :---            | :---                                          |
| REPL            | Read-Eval-Print Loop（プロンプト → 入力 → 実行） |
| `uart_getc`     | シリアル受信（S06 と同様）                    |
| `read_line`     | ラインエディタ（バッファ入力 + バックスペース対応） |
| `strcmp`        | 1 バイトずつ比較して文字列一致を判定          |
| コマンド分岐    | `strcmp` → `je` で各コマンド処理へ分岐        |
| バックスペース  | BS (`0x08`) → スペース → BS で文字を消去     |

**コマンド一覧**:

| コマンド | 動作                         |
| :---     | :---                         |
| `hello`  | `"Hello, 8086!"` と表示      |
| `help`   | コマンド一覧を表示            |
| `quit`   | シェル終了 → `"Bye!"` を表示 |

**空行入力**: 何も表示せずプロンプトに戻る。

### S13: マルチセクタ読み込み

**難易度**: ★★★★★

512 バイトの壁を超え、ディスクから追加セクタを読み込む。BIOS ディスクサービス (INT 0x13) を使用する唯一のシナリオ。

| 学習項目             | 説明                                               |
| :---                 | :---                                               |
| `INT 0x13 AH=02h`    | ディスクからセクタを読み込みメモリに格納           |
| CHS アドレッシング   | Cylinder / Head / Sector でディスク位置を指定      |
| セクタ番号           | 1 始まり（セクタ 2 = オフセット 512 バイト目）     |
| `0x7E00`             | ブートセクタ直後のロード先（0x7C00 + 512）         |
| フラットバイナリ     | NASM `-f bin` + `ORG 0x7C00`（リンカ不使用）       |
| クロスセクタ実行     | `jmp 0x7E00` でセクタ 2 のコードに移行             |

**構造**:

```text
Sector 1 (0x7C00): ブートローダ
  ├── セグメント初期化
  ├── UART で "Loading..." 表示
  ├── INT 0x13 AH=02h（セクタ 2 読み込み, CL=2）
  └── JMP 0x7E00

Sector 2 (0x7E00): 読み込まれたコード
  ├── "Sector 2 loaded!" 表示
  ├── SP レジスタの値表示
  └── HLT
```

### S14: 32-bit プロテクトモード

**難易度**: ★★★★★

16-bit リアルモードから 32-bit プロテクトモードへ移行する。BIOS 割り込みが使えなくなり、全 I/O をポート直接アクセスで行う。

| 学習項目                      | 説明                                                               |
| :---                          | :---                                                               |
| GDT (Global Descriptor Table) | セグメントの属性（Base / Limit / 権限 / タイプ）を定義            |
| ヌルディスクリプタ            | GDT[0] = 0（CPU が要求）                                           |
| コードセグメント (0x08)       | Base=0, Limit=4GB, 32-bit, Exec/Read, Ring 0                       |
| データセグメント (0x10)       | Base=0, Limit=4GB, 32-bit, Read/Write, Ring 0                      |
| `lgdt [gdt_descriptor]`       | GDTR に GDT の位置とサイズをロード                                 |
| CR0.PE ビット                 | `mov eax, cr0` → `or eax, 1` → `mov cr0, eax` でプロテクトモード有効化 |
| ファージャンプ                | `jmp 0x08:pm_entry` でパイプラインをフラッシュ                     |
| セグメントセレクタ            | `mov ax, 0x10` → `mov ds/es/fs/gs/ss, ax`                         |
| 32-bit レジスタ               | EAX, ESI, EDI, ESP（16-bit の 32-bit 拡張版）                      |
| フラットメモリモデル          | Base=0, Limit=4GB（全メモリに線形アクセス）                        |
| I/O ポート継続使用            | `in`/`out` は CPL=0 で引き続き使用可能                             |

**プロテクトモード移行の手順**:

```text
1. CLI                    ← 割り込み禁止
2. LGDT [gdt_descriptor]  ← GDT を CPU に登録
3. MOV EAX, CR0           ← CR0 読み出し
4. OR EAX, 1              ← PE ビットをセット
5. MOV CR0, EAX           ← プロテクトモード有効化
6. JMP 0x08:pm_entry      ← ファージャンプ（16-bit キューをフラッシュ）
7. セグメントレジスタに 0x10（データセレクタ）を設定
   ↑ ここから 32-bit 命令が使える。BIOS 割り込みは使用不可。
```

**GDT の構造（各エントリ 8 バイト）**:

```text
Offset  Size  Field
  0      2    Limit[15:0]
  2      3    Base[23:0]
  5      1    Access Byte (P|DPL|S|Type)
  6      1    Flags (G|D/B|L|AVL) + Limit[19:16]
  7      1    Base[31:24]
```

**2 セクタ構造**: セクタ 1 に 16-bit ブートローダ + GDT、セクタ 2 に 32-bit PM コード。INT 0x13 でセクタ 2 を 0x7E00 にロード。

|              | S01〜S12                       | S13〜S14                             |

### 16550 UART (COM1)

| I/O ポート | レジスタ名 | 説明                                   | アクセス |
| :---       | :---       | :---                                   | :---     |
| `0x3F8`    | THR / RBR  | 送信保持 / 受信バッファ                | W / R    |
| `0x3FD`    | LSR        | ラインステータスレジスタ               | R        |

**LSR フラグビット**:

| ビット | 名前   | 説明                                   |
| :---   | :---   | :---                                   |
| 0      | DR     | Data Ready — 1 = 受信データあり         |
| 5      | THRE   | Transmitter Holding Register Empty — 1 = 送信可能 |

**送信手順**:

1. LSR (0x3FD) の bit 5 (THRE) をチェック
2. THRE = 1 になるまでポーリング (`in al, dx` → `test al, 0x20` → `jz`)
3. THR (0x3F8) に文字を書き込み (`out dx, al`)

**受信手順**:

1. LSR (0x3FD) の bit 0 (DR) をチェック
2. DR = 1 になるまでポーリング (`in al, dx` → `test al, 0x01` → `jz`)
3. RBR (0x3F8) から 1 バイト読み込み (`in al, dx`)

## I/O ルーチン

本教材は文字入出力に 16550 UART（COM1, `0x3F8`）の I/O ポートを直接操作する方針（「ハードウェア情報」の [16550 UART (COM1)](#16550-uart-com1) を参照）。BIOS 割り込み（`INT` 命令）は S13/S14 のディスク読み込みでのみ使用する。

**定義元**: QEMU が読み込む **SeaBIOS ROM**（QEMU i386 のデフォルトファームウェア）。`INT n` 命令を実行すると CPU がリアルモードの割り込みベクタテーブル（IVT, `0x0000`-`0x03FF`）を参照し、対応する SeaBIOS のハンドラへ分岐する。

**レジスタ規約（BIOS 共通）**: `AH` にサービス番号（機能）を設定し、その他のレジスタ（`AL`, `BX`, `CX`, `DX`, `ES:BX` 等）にパラメータを渡す。返り値は `AX`、エラー有無はキャリーフラグ `CF`（`CF=1` でエラー）。

**主な BIOS 割り込み**:

| 割り込み            | 機能             | 入力                                                                                   | 返り値                                       | 教材使用   |
| :---                | :---             | :---                                                                                   | :---                                         | :---       |
| `INT 0x10 AH=0Eh`   | テレタイプ出力   | `AL`=文字, `BH`=ページ番号                                                             | なし                                         | 未使用     |
| `INT 0x13 AH=02h`   | ディスクセクタ読込 | `AL`=セクタ数, `CH`=シリンダ下位, `CL`=セクタ(1-)/シリンダ上位2bit, `DH`=ヘッド, `DL`=ドライブ, `ES:BX`=バッファ | `AH`=ステータス, `CF`=エラー, `AL`=読込セクタ数 | **S13, S14** |
| `INT 0x13 AH=00h`   | ディスクリセット | `DL`=ドライブ                                                                          | `AH`=ステータス, `CF`                        | 未使用     |
| `INT 0x16 AH=00h`   | キーボード入力   | なし                                                                                   | `AH`=スキャンコード, `AL`=ASCII              | 未使用     |
| `INT 0x1A AH=00h`   | システムタイマー読取 | なし                                                                                | `CX:DX`=ティック数, `AL`=midnight flag       | 未使用     |
| `INT 0x19`          | ブートストラップ | なし                                                                                   | 戻らない（再ブート）                         | 未使用     |

**使用例（ディスクセクタ読込）**:

```asm
mov  ah, 0x02        ; 機能: セクタ読込
mov  al, 1           ; 読込セクタ数
mov  ch, 0           ; シリンダ
mov  cl, 2           ; セクタ番号（1 始まり）
mov  dh, 0           ; ヘッド
mov  dl, 0x00        ; ドライブ（フロッピー）
mov  bx, 0x7E00      ; ES:BX = 転送先バッファ
int  0x13            ; 実行。CF=1 ならエラー
```

**注意**: BIOS 割り込みは **リアルモード（16-bit）でのみ使用可能**。IVT はリアルモード専用の構造であり、プロテクトモード（S14 の 32-bit 部）では IDT の再設定が必要なため BIOS を呼べない。教材はブート直後のリアルモード区間でセクタを読み込み、その後プロテクトモードへ移行する。

## 次のステップ

1. 各シナリオのソースを開き、コメントを読みながらコードを追う
2. `make dump S=s02_registers` で逆アセンブル結果を確認し、各命令の機械語を観察する
3. 即値を変更してビルド → 実行し、結果の変化を確認する
4. S01 の `uart_putc` を理解したら、S06 の `uart_getc` と対比して送受信の対称性を学ぶ
5. S12 をベースに新しいコマンド（例: `echo`, `hex`）を追加してみる
6. S13→S14 の流れでブートローダ → プロテクトモード移行の全体像を掴む
