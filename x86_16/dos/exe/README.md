# 8086 アセンブラ DOS .EXE ワークショップ

DOS 向け 16-bit .EXE プログラムを 14 のシナリオで段階的に学ぶ。
NASM（`-f obj`）+ ALINK でビルドし、DOSBox で実行する。

## 前提条件

| ツール | 用途 | インストール例 |
|--------|------|----------------|
| NASM | アセンブラ | `apt install nasm` |
| ALINK | リンカ（OMF → EXE） | [ALINK](http://alink.sourceforge.net/) をダウンロード |
| DOSBox | DOS エミュレータ | `apt install dosbox` |

## ビルドと実行

```bash
# 全シナリオをビルド
make

# 特定のシナリオを DOSBox で実行
make run S=s01_hello

# 逆アセンブル（EXE ヘッダ 512 バイトをスキップ）
make dump S=s01_hello

# 掃除
make clean
```

## .EXE フォーマットの基礎

```text
┌──────────────────┐
|   EXE ヘッダ       |  MZ シグネチャ + 再配置情報
├──────────────────┤
|   コードセグメント |  segment .text
├──────────────────┤
|   データセグメント |  segment .data
├──────────────────┤
|   スタックセグメント|  segment .stack
└──────────────────┘
```

- **COM との違い**: COM は 1 セグメント（64KB）に全て詰める。EXE は複数セグメントを持てる。
- **エントリポイント**: ALINK は `segment .text` の先頭から実行を開始する。
- **DS の初期化**: EXE 起動時、DS はデータセグメントではなく PSP を指す。明示的に `mov ax, seg label / mov ds, ax` が必要。

## シナリオ一覧

### S01: Hello World

**難易度**: ★☆☆☆☆

最も基本的な DOS .EXE プログラム。文字列を表示して終了する。

| 学習項目 | 説明 |
|----------|------|
| `segment` | NASM OBJ 形式のセグメント宣言 |
| DS 初期化 | `mov ax, seg label` / `mov ds, ax` でデータアクセス可能に |
| `INT 21h AH=09h` | `$` 終端文字列の表示 |
| `INT 21h AH=4Ch` | プログラム終了（リターンコード付き） |
| `segment .stack` | スタックセグメントの宣言 |

### S02: レジスタと算術

**難易度**: ★☆☆☆☆

レジスタ間のデータ移動と四則演算。16進数表示サブルーチンを導入。

| 学習項目 | 説明 |
|----------|------|
| `MOV`, `ADD`, `SUB`, `INC`, `DEC` | 基本算術命令 |
| `AH` / `AL` | レジスタの上位・下位バイトアクセス |
| `LODSB` | `[DS:SI]` から 1 バイト読み込み、SI++ |
| `INT 21h AH=02h` | 1 文字出力（DL = 文字コード） |
| `print_hex16` | ニブル→ASCII 変換サブルーチン |

### S03: スタック操作

**難易度**: ★★☆☆☆

スタック（LIFO）の動作を理解する。PUSH/POP でデータを保存・復元するパターンを学ぶ。

| 学習項目 | 説明 |
|----------|------|
| `PUSH` / `POP` | スタックへの保存・復元（SP が ±2） |
| LIFO | Last In, First Out（最後に入れたものが最初に出る） |
| SP の変化 | PUSH で減少、POP で増加 |
| レジスタ保存パターン | `PUSH AX` → 作業 → `POP AX` で値を保護 |

**注意**: 8086 では `PUSH imm`（即値プッシュ）は使用不可。`MOV AX, val` → `PUSH AX` とする。

### S04: ループと条件分岐

**難易度**: ★★☆☆☆

`LOOP` 命令と `CMP` + 条件ジャンプを使った反復処理。

| 学習項目 | 説明 |
|----------|------|
| `LOOP` | CX を自動デクリメントし、CX≠0 ならジャンプ |
| `CMP` + `JLE` | 比較 → 条件付きジャンプ（カウントアップ） |
| `TEST` | AND を実行せずフラグだけ更新（偶数/奇数判定） |

### S05: 文字列操作

**難易度**: ★★☆☆☆

8086 の文字列命令群と `REP` プレフィックスを学ぶ。

| 学習項目 | 説明 |
|----------|------|
| `LODSB` | `[DS:SI]` → AL、SI++ |
| `STOSB` | AL → `[ES:DI]`、DI++ |
| `SCASB` | AL と `[ES:DI]` を比較、DI++ |
| `MOVSB` | `[DS:SI]` → `[ES:DI]`、SI++, DI++ |
| `REP` / `REPNE` | CX 回繰り返し（等しい/等しくない間） |
| `CLD` | ディレクションフラグをクリア（SI/DI 増加方向） |
| 文字列長測定 | `REPNE SCASB` で null ターミネータを探す |
| 文字列コピー | `REP MOVSB` でメモリブロックを一括コピー |

### S06: キーボード入力

**難易度**: ★★★☆☆

ユーザからのキーボード入力を読み取り、画面にエコーバックする。

| 学習項目 | 説明 |
|----------|------|
| `INT 21h AH=01h` | 文字入力（エコーあり）、AL = ASCII コード |
| `INT 21h AH=07h` | 文字入力（エコーなし） |
| `INT 21h AH=0Ah` | バッファ付き行入力 |
| バッファ構造 | `[最大長][実際の長さ][文字列...]` |

### S07: サブルーチン

**難易度**: ★★★☆☆

`CALL` / `RET` による構造化プログラミングと、レジスタ経由のパラメータ渡し。

| 学習項目 | 説明 |
|----------|------|
| `CALL` / `RET` | リターンアドレスをスタックに保存・復元 |
| パラメータ渡し | 呼び出し前にレジスタ（AX, BX, SI 等）に値を設定 |
| レジスタ保存 | `PUSH` → 処理 → `POP` で呼び出し元のレジスタを保護 |
| ネスト呼び出し | サブルーチンからさらにサブルーチンを呼ぶ |

**注意**: 8086 では `PUSHA`/`POPA` は使用不可（80186+）。個別に PUSH/POP する。

### S08: 直接 VRAM 書き込み

**難易度**: ★★★☆☆

DOS を経由せず、ビデオメモリ（`0xB800`）に直接書き込んでカラー文字を表示する。

| 学習項目 | 説明 |
|----------|------|
| VRAM セグメント | `ES = 0xB800`（テキストモード 80×25） |
| キャラクタ + 属性 | `[ASCII][属性]` の 2 バイトで 1 文字を構成 |
| 属性バイト | `[Blink][BG RGB][Bright][FG RGB]`（計 8 ビット） |
| `STOSW` | AX（文字+属性）を `[ES:DI]` に書き込み |
| カラーパレット | 16 色の前景色・8 色の背景色の組み合わせ |

---

### S09: 分岐とメニュー

**難易度**: ★★★★☆

キー入力に応じた分岐でメニューシステムを構築する。符号付き / 符号なし比較の違いも体験。

| 学習項目 | 説明 |
|----------|------|
| `CMP` とフラグ | ZF, CF, SF, OF の変化を観察 |
| 符号付き: `JL` / `JG` | SF ≠ OF のときにジャンプ |
| 符号なし: `JB` / `JA` | CF = 1 のときにジャンプ |
| メニューディスパッチ | キー入力 → `CMP` → `JE` で該当処理へ分岐 |

**ポイント**: `0xFB` は符号付きなら -5、符号なしでは 251。同じビット列でも比較命令により結果が変わる。

### S10: ビット演算

**難易度**: ★★★★☆

AND / OR / XOR / NOT / SHL / SHR を使い、ビット単位の操作を理解する。

| 学習項目 | 説明 |
|----------|------|
| `AND` | マスク（特定ビットの抽出） |
| `OR` | ビットの設定（フラグの組み合わせ） |
| `XOR` | ビットの反転、レジスタのゼロ化（`XOR AX, AX`） |
| `SHL` / `SHR` | 左シフト＝×2、右シフト＝÷2 |
| 2進数表示 | `SHL` で 1 ビットずつ桁上げ → キャリーフラグで 0/1 判定 |

### S11: メモリとブロック操作

**難易度**: ★★★★☆

メモリへのブロック単位の書き込み・コピーと、8086 の多彩なアドレッシングモードを体験。

| 学習項目 | 説明 |
|----------|------|
| `REP STOSB` | メモリブロックのフィル（指定バイトで埋める） |
| `REP MOVSB` | メモリブロックのコピー |
| `[即値]` | `mov byte [buf1+3], 'X'` |
| `[BX+SI]` | ベース + インデックス アドレッシング |

### S12: ミニシェル（総合）

**難易度**: ★★★★★

これまでの全要素を組み合わせた対話型コマンドシェル。

| 学習項目 | 説明 |
|----------|------|
| 行エディタ | 1 文字ずつ入力、Backspace で削除 |
| 文字列比較 | `strcmp` サブルーチン（1 バイトずつ比較） |
| 大文字→小文字 | A-Z を自動で a-z に変換 |
| 画面クリア | BIOS `INT 10h AH=06h` で全画面スクロール |
| コマンド実行 | `hello`, `clear`, `help`, `ver`, `exit` |

**コマンド一覧**:

- `hello` — 挨拶メッセージを表示
- `clear` — 画面をクリア

### S13: ファイル I/O

**難易度**: ★★★★★

DOS のファイル操作 API を使い、ファイルの作成・書き込み・読み込みを行う。

| 学習項目 | 説明 |
|----------|------|
| `INT 21h AH=3Ch` | ファイル作成（CX = 属性、戻り値 AX = ハンドル） |
| `INT 21h AH=40h` | ファイル書き込み（BX = ハンドル、CX = バイト数） |
| `INT 21h AH=3Dh` | ファイルオープン（AL = アクセスモード） |
| `INT 21h AH=3Fh` | ファイル読み込み（戻り値 AX = 実際に読んだバイト数） |
| `INT 21h AH=3Eh` | ファイルクローズ（BX = ハンドル） |
| キャリーフラグ | CF=1 でエラー、CF=0 で成功 |
| ファイルハンドル | DOS が管理するファイル識別子（word 値） |

### S14: PSP とコマンドライン

**難易度**: ★★★★★

Program Segment Prefix（PSP）の構造を理解し、コマンドライン引数を処理する。

| 学習項目 | 説明 |
|----------|------|
| PSP | プログラムの前に配置される 256 バイトの管理領域 |
| DS 初期値 | EXE 起動時、DS = ES = PSP セグメント |
| `PSP:80h` | コマンドテール長（1 バイト） |
| `PSP:81h-FFh` | コマンドテール（スペース + 引数） |
| `PSP:2Ch` | 環境変数ブロックのセグメントアドレス |
| セグメント切り替え | PSP のデータを読むために一時的に DS を切り替え |

**使い方**:

```
S14_PSP.EXE hello world
```

コマンドライン引数 `hello world` が表示される。

## 8086 命令リファレンス

### データ転送

| 命令 | 動作 |
|------|------|
| `MOV dst, src` | dst ← src |
| `PUSH src` | スタックに保存（SP -= 2） |
| `POP dst` | スタックから復元（SP += 2） |
| `XCHG a, b` | a と b を交換 |
| `LEA reg, mem` | 実効アドレスを計算して reg に格納 |

### 算術

| 命令 | 動作 |
|------|------|
| `ADD dst, src` | dst += src |
| `SUB dst, src` | dst -= src |
| `INC dst` | dst++ |
| `DEC dst` | dst-- |
| `MUL src` | AX ← AL × src（8-bit） |
| `DIV src` | AL ← AX ÷ src, AH ← 余り |

### 論理・シフト

| 命令 | 動作 |
|------|------|
| `AND dst, src` | ビット単位の AND |
| `OR dst, src` | ビット単位の OR |
| `XOR dst, src` | ビット単位の XOR |
| `NOT dst` | ビット反転 |
| `SHL dst, 1` | 左に 1 ビットシフト（×2） |
| `SHR dst, 1` | 右に 1 ビットシフト（÷2） |
| `SHL dst, CL` | CL ビット左シフト |
| `SHR dst, CL` | CL ビット右シフト |

### 比較・分岐

| 命令 | 動作 | 条件 |
|------|------|------|
| `CMP a, b` | a - b（フラグだけ更新） | — |
| `TEST a, b` | a AND b（フラグだけ更新） | — |
| `JMP label` | 無条件ジャンプ | — |
| `JE` / `JZ` | 等しいとき | ZF=1 |
| `JNE` / `JNZ` | 等しくないとき | ZF=0 |
| `JL` / `JNGE` | より小さい（符号付き） | SF≠OF |
| `JG` / `JNLE` | より大きい（符号付き） | ZF=0, SF=OF |
| `JB` / `JNAE` | より小さい（符号なし） | CF=1 |
| `JA` / `JNBE` | より大きい（符号なし） | CF=0, ZF=0 |
| `LOOP label` | CX--, CX≠0 ならジャンプ | — |

### 文字列

| 命令 | 動作 |
|------|------|
| `LODSB` | AL ← `[DS:SI]`, SI++ |
| `STOSB` | `[ES:DI]` ← AL, DI++ |
| `MOVSB` | `[ES:DI]` ← `[DS:SI]`, SI++, DI++ |
| `SCASB` | 比較: AL と `[ES:DI]`, DI++ |
| `CMPSB` | 比較: `[DS:SI]` と `[ES:DI]`, SI++, DI++ |
| `REP` | CX 回繰り返す |
| `REPZ` / `REPE` | 等しい間、CX 回繰り返す |
| `REPNE` / `REPNZ` | 等しくない間、CX 回繰り返す |

### 制御

| 命令 | 動作 |
|------|------|
| `CALL label` | 次のアドレスを PUSH してジャンプ |
| `RET` | スタックから POP してジャンプ |
| `CLC` / `STC` | キャリーフラグのクリア / セット |
| `CLD` / `STD` | ディレクションフラグのクリア / セット |
| `CLI` / `STI` | 割り込み禁止 / 許可 |
| `HLT` | CPU 停止 |
| `NOP` | 何もしない |

## DOS 割り込みリファレンス

### INT 21h — DOS サービス

| AH | 機能 | 入力 | 出力 |
|----|------|------|------|
| `01h` | 文字入力（エコーあり） | — | AL = ASCII |
| `02h` | 文字出力 | DL = 文字 | — |
| `07h` | 文字入力（エコーなし） | — | AL = ASCII |
| `09h` | 文字列出力 | DS:DX → `$` 終端文字列 | — |
| `0Ah` | バッファ行入力 | DS:DX → バッファ | バッファに格納 |
| `3Ch` | ファイル作成 | DS:DX=ファイル名, CX=属性 | AX=ハンドル, CF=エラー |
| `3Dh` | ファイルオープン | DS:DX=ファイル名, AL=モード | AX=ハンドル, CF=エラー |
| `3Eh` | ファイルクローズ | BX=ハンドル | CF=エラー |
| `3Fh` | ファイル読み込み | BX=ハンドル, CX=バイト数, DS:DX=バッファ | AX=読み取り数, CF=エラー |
| `40h` | ファイル書き込み | BX=ハンドル, CX=バイト数, DS:DX=データ | AX=書き込み数, CF=エラー |
| `4Ch` | プログラム終了 | AL=リターンコード | — |

### INT 10h — BIOS ビデオサービス

| AH | 機能 | 入力 |
|----|------|------|
| `00h` | ビデオモード設定 | AL = モード番号（3 = 80×25 テキスト） |
| `02h` | カーソル位置設定 | BH = ページ, DH = 行, DL = 列 |
| `06h` | 上スクロール | AL = 行数（0=全画面）, BH = 属性, CX/DX = 範囲 |
| `0Eh` | テレタイプ出力 | AL = 文字, BH = ページ |

### INT 16h — BIOS キーボードサービス

| AH | 機能 | 出力 |
|----|------|------|
| `00h` | キー入力待ち | AL = ASCII, AH = スキャンコード |
| `01h` | キー入力チェック | ZF=0 で入力あり |

## 8086 vs 80186+ の違い

このチュートリアルのコードはすべて **厳密な 8086** 互換である。

| 命令 | 8086 | 80186+ | 本チュートリアルでの対処 |
|------|------|--------|--------------------------|
| `PUSH imm` | ✗ | ○ | `MOV AX, val` → `PUSH AX` |
| `PUSHA` / `POPA` | ✗ | ○ | 個別 `PUSH`/`POP` |
| `SHL reg, N` (N>1) | ✗ | ○ | `MOV CL, N` → `SHL reg, CL` |
| `ENTER` / `LEAVE` | ✗ | ○ | `PUSH BP` / `MOV BP, SP` |

## 次のステップ

- 各シナリオのソースを開き、コメントを読みながらコードを追う
- `make dump S=s02_registers` で機械語を確認し、命令がどうエンコードされるか観察する
- 値を変更してビルド → 実行し、結果の変化を確認する
- S12 をベースに新しいコマンド（例: `echo`, `count`）を追加してみる
- S14 を拡張してコマンドライン引数のパーサーを作ってみる
